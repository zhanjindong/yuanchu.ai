一、算法到底是什么？

你打开 IDE，开始写代码。循环嵌套循环，变量换来换去，代码跑起来了，结果看起来对——但你心里有没有闪过一丝不安：

「这段代码到底快不快？如果数据量扩大100倍，它会不会挂？」

如果你的答案是「不知道」或者「应该没问题吧」，那这篇文章就是为你写的。

算法，就是那个让你从「凭感觉写代码」变成「用脑子写代码」的东西。它不神秘，也不高大上——本质上就是「用更聪明的方式解决问题」。

读完这篇文章，你会掌握算法的核心思维，理解时间复杂度、空间复杂度这些面试必考概念，了解排序、查找、图算法、动态规划这些最常用的算法套路。最重要的是，你会获得一种新能力：写完代码后，你可以有理有据地说「这段代码够快」或者「这里需要优化」。

二、时间复杂度与空间复杂度

评价一个算法好坏，最重要的两个指标是：

  • 时间复杂度：算法运行需要多长时间
  • 空间复杂度：算法运行需要多少内存

你不需要记住具体怎么算，只需要理解「随着数据量增长，算法性能会怎么变化」。

1. 大O表示法：描述增长趋势

计算机科学家用「大O表示法」来描述算法复杂度。常见的有这些：

复杂度名称当数据量扩大10倍…典型场景
O(1)常数时间不变数组按下标访问
O(log n)对数时间慢一点二分查找
O(n)线性时间慢10倍遍历数组
O(n log n)线性对数时间慢约10倍快速排序、归并排序
O(n²)平方时间慢100倍冒泡排序、双重循环
O(2ⁿ)指数时间慢1024倍递归fibonacci

你可以这样理解：复杂度就是描述「最坏情况下，操作次数会随着数据量怎么增长」。

实战案例

假设你有 1000 条数据：

  • O(n) = 1000 次操作
  • O(n²) = 1,000,000 次操作
  • O(log n) ≈ 10 次操作

所以为什么面试官总是问「你这个算法复杂度是多少」——因为数据量大了之后，复杂度低的算法能把复杂度高的按在地上摩擦。

2. 空间复杂度：内存怎么花

除了时间，还要考虑内存占用：

空间复杂度含义例子
O(1)固定内存，不随数据量变化遍历数组
O(n)内存随数据量线性增长创建一个新数组存结果
O(n²)内存随数据量平方增长二维数组

记住一个原则：能用时间换空间的算法，往往更实用；能用空间换时间的算法，往往更快。具体怎么选，要看你的场景——是计算资源紧张，还是内存紧张。

一句话理解

复杂度就是描述「数据量变大时，算法会慢多少」的数学语言。O(1) 是最快的大哥，O(2ⁿ) 是最慢的弟中弟。

三、排序算法：从小到大

排序是算法中最经典的问题。10个最常见的算法中，至少有3个是排序——因为排序太常用了，而且它还能引出很多重要的算法思想。

1. 冒泡排序：最直观的「菜鸟算法」

冒泡排序的名字很形象——像水底的气泡一样，大的元素慢慢「冒」到右边。

算法思路：

  1. 从左到右比较相邻两个元素
  2. 如果左边比右边大，就交换
  3. 重复 n 遍，最后最大的元素就「冒」到了最右边
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# 复杂度：O(n²) —— 很慢，但容易理解

2. 快速排序：快的代名词

快速排序是实际使用中最常见的排序算法——「快速」两个字不是白叫的。

核心思想：分而治之

  1. 选一个「基准元素」（pivot）
  2. 把数组分成两部分：左边的都比基准小，右边的都比基准大
  3. 对左右两部分分别递归执行上述步骤
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return quick_sort(left) + middle + quick_sort(right)

# 平均复杂度：O(n log n)
反例警示

快速排序有一个致命弱点：如果每次都选到最大或最小的元素作为基准，时间复杂度会退化成 O(n²)。

解决办法：随机选择基准，或者用「三数取中」法。

3. 排序算法对比

算法时间复杂度空间复杂度稳定性适用场景
冒泡排序O(n²)O(1)稳定教学演示
快速排序O(n log n)O(log n)不稳定通用场景
归并排序O(n log n)O(n)稳定需要稳定排序
堆排序O(n log n)O(1)不稳定需要原地排序
实战建议

Python 内置的 sorted() 和 list.sort() 用的都是 Timsort——一种结合了归并排序和插入排序的「超级排序」。

四、查找算法：大海捞针

1. 线性查找：最笨的方法

从头遍历到尾，一个一个找——这是最直观的办法。

def linear_search(arr, target):
    for i, x in enumerate(arr):
        if x == target:
            return i
    return -1

# 复杂度：O(n)

2. 二分查找：智慧的搜索

二分查找是算法中最经典的「智慧」体现——每次排除一半的数据，效率极高。

前提条件：数组必须有序

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1

# 复杂度：O(log n)
二分查找示例：在19个元素中找9

3. 哈希查找：瞬间定位

哈希查找就是「更快」——理论上可以做到 O(1)，也就是常数时间。

# Python 的字典就是哈希表
score = {"Alice": 95, "Bob": 88, "Charlie": 92}
print(score["Alice"])  # 95 —— 瞬间返回！

# 复杂度：O(1)
延伸思考

掌握了二分查找，你就掌握了「如何快速缩小搜索范围」这个核心思维——这是很多高级算法的基础。

五、图算法：关系的艺术

现实世界充满了「关系」：社交网络、道路交通、网页链接——这些都是图（Graph）。

1. 图的基本概念

  • 顶点（Vertex）：图中的节点
  • 边（Edge）：顶点之间的关系
  • 有向图：边有方向
  • 无向图：边无方向

2. BFS 与 DFS：图的遍历

遍历图有两种核心方式：

  • BFS（广度优先搜索）：一层一层地扩散，像水面波纹
  • DFS（深度优先搜索）：一条路走到黑，再回溯
from collections import deque

# BFS - 广度优先搜索
def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    
    while queue:
        node = queue.popleft()
        print(node, end=' ')
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

# DFS - 深度优先搜索  
def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    
    visited.add(start)
    print(start, end=' ')
    
    for neighbor in graph[start]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)

# 复杂度：O(V + E)，V是顶点数，E是边数

3. 最短路径：Dijkstra 算法

找最短路径最经典的算法是Dijkstra 算法——它解决了「从A点到B点，怎么走最近」这个问题。

import heapq

def dijkstra(graph, start, end):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]
    
    while pq:
        current_dist, current = heapq.heappop(pq)
        
        if current == end:
            return distances[end]
        
        if current_dist > distances[current]:
            continue
        
        for neighbor, weight in graph[current]:
            distance = current_dist + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))
    
    return distances[end]

# 复杂度：O((V + E) log V)
实战案例

地图导航、网络路由、游戏寻路——这些背后都是最短路径算法。你每次用导航软件规划路线，背后都跑着 Dijkstra 或者它的改进版本 A*。

六、动态规划：聪明的取舍

动态规划（Dynamic Programming，简称 DP）是算法中最「聪明」的思想——它用空间换时间，用记忆化避免重复计算。

1. 斐波那契数列：理解动态规划的起点

先看看递归版本的斐波那契数列：

# 暴力递归 - 指数级复杂度！
def fib_bad(n):
    if n <= 1:
        return n
    return fib_bad(n-1) + fib_bad(n-2)

# 问题：计算 fib(40) 要计算 10 亿+ 次！
# 原因：大量重复计算

问题在于：fib(5) = fib(4) + fib(3)，而 fib(4) 和 fib(3) 都被计算了无数次。

2. 用动态规划优化

# 方法1：自底向上 + 数组记忆
def fib_dp1(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# 方法2：空间优化（只需要前两个值）
def fib_dp2(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr

# 复杂度：O(n) 时间，O(1) 空间！
核心洞察

动态规划的本质：把一个复杂问题拆成小问题，但小问题的答案要存下来下次再用——这就叫「最优子结构」和「重叠子问题」。

3. 经典 DP 问题

动态规划能解决的问题都有两个特征：

  • 最优子结构：大问题可以拆成小问题
  • 重叠子问题：小问题会被重复计算
问题描述核心思路
0-1背包物品重量和价值，求最大价值选或不选第i个物品
最长公共子序列两个字符串的最长公共部分逐字符比较
股票买卖最佳买卖时机记录最低价格和最大利润
爬楼梯到第n级有多少种走法fib(n) = fib(n-1) + fib(n-2)
常见误区

很多人觉得动态规划难，其实难的不是算法本身，而是「能不能把问题拆成子问题」。这一步才是真正需要练习的。

七、总结与下一步

好，我们已经走完了算法的核心基础知识。让我帮你梳理一下：

  • 算法是解决问题的步骤——用更聪明的方式，而不是蛮力
  • 复杂度是算法的标尺——O(1) 最快，O(2^n) 最慢，数据量大了差距惊人
  • 排序算法——快速排序最常用，归并排序最稳定
  • 查找算法——二分查找是神器，哈希表是终极方案
  • 图算法——BFS/DFS 是基础，Dijkstra 解决最短路径
  • 动态规划——用空间换时间，解决重复计算问题

我的看法：算法不是「面试才要会」的东西，而是「日常编程的基本功」。你不需要背代码，但你需要理解「为什么这个算法比那个快」。这种思维一旦养成，你会发现看问题的角度都不一样了。

下一步建议：

  1. 刷一道经典算法题——LeetCode 第 1 题「两数之和」，用不同方法实现并比较复杂度
  2. 手写一次快速排序——不要复制，闭卷写，感受分治思想的精髓
  3. 理解递归的本质——动态规划的核心，理解递归就理解了一半的 DP
最后一句话

算法不是魔法，它只是「把问题想清楚」的艺术。学会了，你就拥有了程序员最核心的竞争力。
