搜索评测指标全解析

从 Recall@K 到在线 CTR，系统性掌握搜索质量评估

搜索系统好不好，不能凭感觉，得用数据说话。本文系统性介绍搜索系统中从离线到在线的全链路评估指标，覆盖召回、粗排、精排各阶段的考核重点。

一、召回阶段： Recall@K 与召回率

1.1 Recall@K

Recall@K 是召回阶段最核心的指标，衡量的是"Top K 结果中召回多少相关文档"。

Recall@K = (Top K 结果中相关文档数) / (全部相关文档总数)

举例：假设总共有 50 个相关文档，系统返回了 100 条，Top 10 中有 8 个相关的，则 Recall@10 = 8/50 = 16%。

💡 一句话记住

Recall@K 的"K"是截断点——截到第K个，召回率是多少。

1.2 召回率（Recall）

召回率是更广义的指标，通常指"所有相关文档被召回的比例"——相当于 Recall@all。

Recall = 召回的相关文档数 / 总相关文档数

1.3 其他召回指标

指标含义适用场景
Recall@KTop K 召回率离线评估
Hit Rate@KTop K 中是否命中开源检索
Recall@K / Precision@K召回/精确平衡多路召回调优
📊 关键区别

Recall 关注"找全了没有"，Precision 关注"找对了没有"。召回是后者的分子，精确是前者的分子。

二、排序阶段： MRR、nDCG、MAP

2.1 MRR（Mean Reciprocal Rank）

MRR 衡量"第一个相关文档出现在第几位"。只看第一名，是搜索排序最直观的指标。

MRR = 1/n × Σ(1/rank_i) // rank_i = 第i个query第一个相关文档的位置

举例：查询"手机"第一个相关结果是第3位，则该项得分为 1/3；查询"电脑"第一个相关结果是第1位，得分为1。MRR = (1/3 + 1)/2 = 0.667。

💡 一句话记住

MRR 只看"第一个"相关结果在哪——排得越靠前，分数越高。

2.2 nDCG（Normalized Discounted Cumulative Gain）

nDCG 是排序质量的黄金标准，考虑了"相关性权重"——相关度越高的文档越靠前，分数越高。

DCG@K = Σ(rel_i / log2(i+1)) // rel_i为第i位的相关性分数 nDCG@K = DCG@K / IDCG@K // IDCG为理想排序的DCG

举例：假设理想排序应该是[3,2,1,0]（相关度递减），但系统返回了[2,3,1,0]，nDCG 会量化这个差距。

相关性分数含义
3完全相关
2部分相关
1边缘相关
0不相关
⚠️ 踩坑记录

nDCG 需要人工标注相关性——标注质量直接决定指标可信度。标注一致性比指标本身更重要。

2.3 MAP（Mean Average Precision）

MAP 衡量"每个相关文档的排序位置"，综合考量召回和排序。

AP = Σ(Precision@K × rel_k) / 总相关数 MAP = 1/n × Σ(AP_i)

举例：查询返回[相关,相关,不相关,相关]，则 AP = (1/2 + 2/4 + 0 + 3/4)/3 = 0.67。

三、在线指标：点击率、有点比、CTR

3.1 CTR（Click-Through Rate）

点击率是最直接的在线效果指标，衡量"用户点不点"。

CTR = 点击次数 / 曝光次数

3.2 有点比（UVR - User Valid Rate）

有点比衡量"点进来的用户占比"，排除机器流量，更真实。

有点比 = 点击用户数 / 曝光用户数

3.3 文档点击率（Document CTR）

文档点击率关注"每个结果被点的概率"，评估单条结果的质量。

文档 CTR = 文档点击数 / 文档曝光数

3.4 其他在线指标

指标含义用途
UV_CTR独立用户点击率去重
Skip Rate跳过率首条效果
Long Click Rate长尾点击内容深度
Conversions转化次数商业目标
💡 一句话记住

离线指标用来"调模型"，在线指标用来"看效果"——前者是实验数据，后者是真实用户用脚投票。

四、综合指标与实践

4.1 F1 Score

F1 是精确率和召回率的调和平均，兼顾"找全"和"找准"。

F1 = 2 × Precision × Recall / (Precision + Recall)

4.2 MAPE（Mean Absolute Percentage Error）

MAPE 是预测误差指标，常用于搜索结果的"应该排第几"预测。

MAPE = 1/n × Σ(|预测-实际| / 实际)

4.3 指标选用建议

阶段核心指标辅助指标
召回Recall@KHit Rate、召回路数
粗排MRRRecall@K、延迟
精排nDCG、MAP离线AUC
全链路在线CTR有点比、Skip Rate
⚠️ 常见误区
  • 只调离线指标——离线涨分但在线下跌是常态
  • 只看 CTR——短期CTR陷阱可能伤害长尾
  • 指标太多——聚焦核心指标，其他辅助看

五、总结：指标选用逻辑

搜索系统评估的核心逻辑：

  1. 召回阶段：Recall@K 衡量"相关文档召回多少"，目标是"不漏"
  2. 粗排阶段：MRR 衡量"Top结果够不够准"，目标是"减压"
  3. 精排阶段：nDCG/MAP 衡量"排序质量够不够好"，目标是"精准"
  4. 上线后：CTR/有点比 衡量"用户买不买账"，目标是"有效"
📊 一句话记住

召回看"全"，排序看"准"，在线看"转"——三个阶段三重考核。
